Critical reflection on my first(and second) graph

As I have mentioned in my second blog, data visualisation is about taking in data, usually data that is big, and making it easier to understand and interesting to look at. Data visualisation is about exploration and communication, the function of the developers of data visualisation is to make sure that they clearly communicate the data and make sure that they represent it in a way that is easy to understand, and they do this by using signifiers that are culturally common.

In my attempt of data visualisation, I created graphs that ‘analyse’ the most common symptoms of Coronavirus aka covid-19. I found this information on a website called Flatten The Curve, which is a site that provides information to users about how they can take care of themselves and others during this pandemic.

The first graph that I created is a very simple one, it has the x-axis which represent the different symptoms. Instead of putting the names of the different symptoms on the x-axis, I chose to put them next to the dots that represent different symptoms. The reason why I did this was because I wanted to make it easier for my users to read the data, and also for them to save time.

I also used the x-axis to represent the level of commonality of a symptoms in percentages, the higher the percentage the more common the symptom is. I also used colour to represent risk/commonality, the more saturated the colour the more common the symptom. In the first graph I used colour in the text, specifically in the names of the symptoms. In the second graph, I used colour in both the names of the symptoms and the circles that represent the symptoms. I also used the sizes of the circles to represent the commonality of the symptom.
